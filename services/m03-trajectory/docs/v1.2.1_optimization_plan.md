## 2. 优化方案

### 2.1 算法并行化 (预期提升30%)

#### 目标函数
- `detect_stay_points()`
- `calculate_smooth_speed()`
- `filter_gps_drift()`

#### 实现方案
```go
// 使用goroutine并行处理不同用户的数据
func BatchProcess(users []User) {
    var wg sync.WaitGroup
    sem := make(chan struct{}, runtime.NumCPU())
    
    for _, user := range users {
        wg.Add(1)
        sem <- struct{}{}
        
        go func(u User) {
            defer wg.Done()
            defer func() { <-sem }()
            
            processUser(u)
        }(user)
    }
    
    wg.Wait()
}
```

#### 预期效果
- 多用户场景: QPS 1,847 → 2,400 (+30%)
- 单用户延迟: 保持不变

### 2.2 SIMD优化卡尔曼滤波 (预期提升50%)

#### 方案
使用Go的SIMD指令优化向量运算

```go
// 使用github.com/minio/simdjson-go或类似库
// 或手写AVX2/SSE优化

// 快速EWMA替代完整卡尔曼滤波
func FastKalman(values []float64) []float64 {
    result := make([]float64, len(values))
    alpha := 0.3
    result[0] = values[0]
    
    for i := 1; i < len(values); i++ {
        result[i] = alpha*values[i] + (1-alpha)*result[i-1]
    }
    
    return result
}
```

#### 预期效果
- GetSpeed P99: 8.5ms → 5ms (-41%)

### 2.3 零分配优化 (预期减少GC 40%)

#### 方案
```go
// 对象池复用
type PointPool struct {
    pool sync.Pool
}

func (p *PointPool) Get() []GPSPoint {
    return p.pool.Get().([]GPSPoint)[:0]
}

func (p *PointPool) Put(points []GPSPoint) {
    if cap(points) <= 10000 {
        p.pool.Put(points)
    }
}
```

#### 优化点
1. GPSPoint切片池化
2. 减少临时对象分配
3. 预分配map容量

#### 预期效果
- GC频率: -40%
- 内存占用: 312MB → 250MB
