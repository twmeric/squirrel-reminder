## 2. 优化方案详解

### 2.1 算法并行化 ⭐ 优先级P0

**问题**: 单用户处理串行，CPU利用率低

**方案**:
```go
// 并发处理多个用户
func BatchProcess(users []User) {
    sem := make(chan struct{}, runtime.NumCPU())
    var wg sync.WaitGroup
    
    for _, user := range users {
        wg.Add(1)
        sem <- struct{}{}
        go func(u User) {
            defer wg.Done()
            defer func() { <-sem }()
            processUser(u)
        }(user)
    }
    wg.Wait()
}
```

**预期效果**: QPS 1,847 → 2,400 (+30%)

**工作量**: 2天

---

### 2.2 SIMD加速速度计算 ⭐ 优先级P0

**问题**: 卡尔曼滤波纯Go实现，未利用SIMD

**方案**: 使用快速EWMA近似替代完整卡尔曼
```go
func FastSmooth(values []float64) []float64 {
    result := make([]float64, len(values))
    alpha := 0.3
    result[0] = values[0]
    
    for i := 1; i < len(values); i++ {
        result[i] = alpha*values[i] + (1-alpha)*result[i-1]
    }
    return result
}
```

**预期效果**: GetSpeed P99 8.5ms → 5ms (-41%)

**工作量**: 3天

---

### 2.3 零分配优化 ⭐ 优先级P1

**问题**: 大量临时对象分配，GC频繁

**方案**:
```go
type PointPool struct {
    pool sync.Pool
}

func (p *PointPool) Get() []GPSPoint {
    return p.pool.Get().([]GPSPoint)[:0]
}

func (p *PointPool) Put(points []GPSPoint) {
    if cap(points) <= 10000 {
        p.pool.Put(points)
    }
}
```

**预期效果**: 
- GC频率 -40%
- 内存 312MB → 250MB

**工作量**: 2天
